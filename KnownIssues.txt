Version 2.0 - 13/6/26
Modernized the old GitHub project version for continued development:
- Removed names, IDs, and personal sample data from source, docs, and SQL samples.
- Reorganized Java classes into packages by responsibility: app, zoo, animals, people, tickets, promotions, data, services, models, common, and exceptions.
- Added a richer MySQL schema with enclosures, worker assignments, feeding schedules, medical records, and ticket revenue support.
- Added repository and service layers so database access and business rules are separated from the console menu.
- Added console menu options for animal CRUD, worker assignments, feeding scheduling/completion, medical records, and revenue reports.
- Replaced hardcoded database/admin credentials with environment variables or Java system properties.
- Added password hashing for worker/admin login data.
- Added JUnit tests for capacity validation and ticket revenue report validation.
- Renamed ReadMe.txt to README.md and added ER diagram, screenshot preview, setup instructions, and testing instructions.

Known issues / setup notes:
- DB-backed menu options require MySQL to be running and the schema/sample data loaded with DataBase.sql and sample.sql.
- Local Homebrew MySQL may fail to start if an old MySQL 8.3 data directory is used with MySQL 9.6; migrate/dump the old data or reinitialize the local MySQL data directory.
- Maven currently references the local MySQL connector jar from lib/, which works for this project but shows a portability warning.
